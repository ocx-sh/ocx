# Increment 10: cargo-dist Initialization + Base Release Workflow

**Status**: Not Started
**Completed**: —
**ADR Phase**: 5 (steps 14-15)
**Depends on**: Increment 01 (workspace version must be set)

---

## Goal

Initialize cargo-dist for binary builds, archive packaging, and GitHub Release creation. Generate the base `release.yml` workflow.

## Tasks

### 1. Install cargo-dist

```bash
cargo install cargo-dist
```

### 2. Initialize cargo-dist

```bash
cargo dist init
```

Interactive prompts:
- **CI provider**: GitHub Actions only
- **Installers**: None (OCX uses custom install scripts — do NOT enable shell/PowerShell installers)
- **Targets**: All 8 targets from the ADR target matrix

### 3. Configure `dist-workspace.toml`

After `cargo dist init`, review and adjust the generated config:

```toml
[dist]
targets = [
  "x86_64-unknown-linux-gnu",
  "aarch64-unknown-linux-gnu",
  "x86_64-unknown-linux-musl",
  "aarch64-unknown-linux-musl",
  "x86_64-apple-darwin",
  "aarch64-apple-darwin",
  "x86_64-pc-windows-msvc",
  "aarch64-pc-windows-msvc",
]
unix-archive = ".tar.xz"
windows-archive = ".zip"
```

Key settings:
- **No installers** — `installers = []` or omit entirely
- **Unix archives**: `.tar.xz` (not `.tar.gz`)
- **Windows archives**: `.zip`

### 4. Review generated `release.yml`

cargo-dist generates `.github/workflows/release.yml`. Review it for:
- Correct trigger: `on: push: tags: ["v*"]` (or similar tag pattern)
- Build matrix matches all 8 targets
- Archive format is `.tar.xz` for Unix, `.zip` for Windows
- Checksums are generated (`sha256sum.txt`)
- GitHub Release is created with all artifacts

### 5. Handle cargo-dist cross-compilation limitations

Per ADR: `aarch64-unknown-linux-musl` and `aarch64-pc-windows-msvc` have known cargo-dist cross-compilation issues ([#1581](https://github.com/axodotdev/cargo-dist/issues/1581)).

**Strategy**: Start with all 8 targets in cargo-dist config. If cargo-dist fails on specific targets during the first test release, add custom `actions-rust-cross` jobs for those targets (keeping the existing pattern from `deploy-canary.yml`).

### 6. Test with `cargo dist plan`

```bash
cargo dist plan
```

Verify the plan shows all 8 targets with correct archive formats. This doesn't build anything — it just validates the configuration.

## Notes

- cargo-dist generates the workflow — do NOT hand-write it. Modifications go through `cargo dist init` or post-generation edits.
- The generated workflow will need modifications in subsequent increments (release notes, version verification, post-release bump).
- **Do NOT enable cargo-dist installers.** This is a critical constraint from the ADR. OCX uses custom bootstrap scripts.
- If cargo-dist adds `[profile.dist]` to Cargo.toml, that's fine — it optimizes release builds.
