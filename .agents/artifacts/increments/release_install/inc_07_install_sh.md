## Goal

Create the Unix/macOS install script that bootstraps OCX by downloading a binary from GitHub Releases, then using OCX to install itself into its own store.

## Tasks

### 1. Create `website/src/public/install.sh`

The script implements the bootstrap flow from ADR Section 5:

1. **Detect platform**: `uname -s` / `uname -m`
2. **Map to Rust target triple**: e.g., `Linux` + `x86_64` → `x86_64-unknown-linux-gnu`
3. **Query GitHub API**: `https://api.github.com/repos/ocx-sh/ocx/releases/latest` for asset URLs
4. **Download**: correct `.tar.xz` + `sha256sum.txt` to `$TMPDIR`
5. **Verify SHA256**: `sha256sum --check` (or `shasum -a 256` on macOS)
6. **Extract**: bootstrap binary from archive
7. **Run**: `$TMPDIR/ocx install ocx --select` (OCX installs itself)
8. **Remove**: bootstrap binary from `$TMPDIR`
9. **Create `~/.ocx/env`**: POSIX shell bootstrap file (see ADR Section 5 for exact content)
10. **For Fish**: create `~/.config/fish/conf.d/ocx.fish` (see ADR Section 5)
11. **Detect shell + modify profile**:
    - bash: `~/.bash_profile` (fallback `~/.profile`)
    - zsh: `~/.zshenv`
    - fish: no profile edit needed (conf.d is auto-sourced)
12. **Idempotent**: check if source line already exists before appending
13. **Honor opt-out**: `--no-modify-path` flag / `OCX_NO_MODIFY_PATH=1`
14. **Print success message**

### 2. Error handling

Clear error messages for:
- `curl` or `wget` not found
- GitHub API unreachable
- Checksum mismatch
- Unsupported platform (e.g., FreeBSD, 32-bit)
- `ocx install` failure
- Write permission issues

### 3. Shell env file content

**`~/.ocx/env`** (POSIX):
```sh
#!/bin/sh
# OCX environment — generated by install.sh
_ocx_home="${OCX_HOME:-$HOME/.ocx}"
export PATH="${_ocx_home}/installs/ocx.sh/ocx/current/bin:$PATH"
_ocx_bin="${_ocx_home}/installs/ocx.sh/ocx/current/bin/ocx"
if [ -x "$_ocx_bin" ]; then
  eval "$("$_ocx_bin" shell completion 2>/dev/null)" 2>/dev/null || true
fi
unset _ocx_home _ocx_bin
```

**`~/.config/fish/conf.d/ocx.fish`**:
```fish
set -l _ocx_home (set -q OCX_HOME; and echo $OCX_HOME; or echo $HOME/.ocx)
fish_add_path --path "$_ocx_home/installs/ocx.sh/ocx/current/bin"
set -l _ocx_bin "$_ocx_home/installs/ocx.sh/ocx/current/bin/ocx"
if test -x "$_ocx_bin"
  "$_ocx_bin" shell completion --shell fish 2>/dev/null | source
end
```

### 4. Key design constraints (from ADR)

- **Static PATH, dynamic completions**: PATH uses static export (zero exec cost). Only completions invoke the binary.
- **`OCX_HOME`-aware**: Reads `$OCX_HOME` env var, defaults to `~/.ocx`.
- **No regeneration needed**: The `current` symlink path is stable by design.
- **Archive must contain `bin/ocx`**: The `~/.ocx/env` file resolves at `$OCX_HOME/installs/ocx.sh/ocx/current/bin/ocx`.
