## Goal

Ensure GitHub Releases have changelog-based release notes and that Cargo.toml version matches the pushed tag.

## Implementation

### Release Notes

cargo-dist auto-detects `CHANGELOG.md` and extracts the matching version section for `ANNOUNCEMENT_BODY`. Since `task release:prepare` already runs `git-cliff --tag <version> -o CHANGELOG.md` before tagging, the changelog is populated when the release workflow runs. **No git-cliff action in CI needed** — cargo-dist handles it natively.

### Version Verification

Implemented as a reusable workflow (`.github/workflows/verify-version.yml`) referenced via `plan-jobs = ["./verify-version"]` in `dist-workspace.toml`. This keeps the generated `release.yml` untouched — cargo-dist wires in the custom job automatically, blocking builds if verification fails.
